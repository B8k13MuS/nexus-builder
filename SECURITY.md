# Nexus-Builder: модель угроз и текущие меры защиты

Актуально на: 21.06.2026 (после исправления code_executor).

## Поверхность атаки

Единственная точка входа — Telegram-бот (`bot.py`). Пользователь отправляет
текст или голосовое сообщение → STT → LLM генерирует код/команды/запросы →
агент их выполняет через `core/tools/*`.

Это значит: **любой ввод пользователя в итоге может стать кодом или shell-
командой**, сгенерированной LLM. Защита должна работать на двух уровнях —
кто может писать боту, и что боту разрешено выполнять.

## Уровень 1: кто может писать боту

| Механизм | Файл | Статус |
|---|---|---|
| Whitelist `ALLOWED_USER_IDS` | `core/security.py`, `config.py` | ✅ работает, но открыт всем по умолчанию, если не задан в `.env` |
| Rate limiting (10 запросов / 300 сек по умолчанию) | `core/security.py` | ✅ |
| Уведомление владельца о попытках доступа | `bot.py` | ✅ |
| Блокировка prompt-injection (статические + самообучаемые паттерны) | `core/security.py` | ✅ |

**Действие для тебя:** убедись, что `ALLOWED_USER_IDS` и `OWNER_USER_ID`
реально заданы в `.env` на сервере — иначе уровень 1 не работает.

## Уровень 2: что разрешено выполнять

| Инструмент | Риск | Защита |
|---|---|---|
| `file_writer` | Низкий — пишет только в `output/` | Не проверялось отдельно в этом аудите |
| `web_search` | Низкий | — |
| `code_executor` | Высокий — произвольный Python | ✅ AST-песочница (исправлено 21.06.2026, см. CHANGES.md 2.2) |
| `shell_executor` | Высокий — произвольная shell-команда | ✅ Белый список бинарей + ограничение рабочей директории |
| `project_builder` | Средний — генерирует файлы проекта | Опирается на `file_writer`, отдельно не аудировался |

## Известные ограничения (не уязвимости, но стоит знать)

- AST-песочница `code_executor` блокирует известные техники обхода, но
  не является полной изоляцией (нет cgroups/контейнера) — лимит только по
  времени (10 сек). Для по-настоящему враждебного кода (например,
  потребляющего много памяти/CPU в рамках разрешённых конструкций)
  изоляции на уровне ОС нет.
- `shell_executor` доверяет LLM в генерации команды — белый список бинарей
  защищает от произвольных программ, но не от вредоносных *аргументов*
  разрешённым программам (например, `pip install` с произвольным пакетом
  из PyPI). Это осознанный компромисс, не баг.
